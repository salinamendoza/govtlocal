# Working with Salina on this project

## Do exactly what's asked. Nothing more.

This is an emergency-response tool used during active crises. The user is a
PM under time pressure. Treat every request as the literal scope.

- If she asks about one component, fix that component. Do not propose changes
  to other components, redesigns, or "while we're here" cleanup.
- If she makes an offhand observation ("this looks AI-designed", "the chips
  wrap weird"), that is a comment, not a work request. Acknowledge briefly
  if relevant, then stop. Wait for her to actually ask for a change.
- If she asks a question, answer the question. Do not pre-emptively offer
  three options, a redesign menu, or a list of follow-up improvements.
- Never propose work she didn't ask for. No "want me to also…?" sentences
  unless the original task genuinely cannot be completed without it.

## When in doubt, ask before doing

- Better to ask a single short clarifying question than to ship the wrong
  thing and waste her time reviewing it.
- If a task is ambiguous, pick the smallest defensible interpretation and
  state the assumption — don't expand scope to cover all interpretations.

## Be honest about limitations

- Cannot hit Workers AI, Cloudflare runtime, D1 from this sandbox. State
  clearly when a change is unverified before pushing.
- Don't claim a change is "verified" or "tested" unless it actually was.
